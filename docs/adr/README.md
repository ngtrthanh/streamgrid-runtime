# Architectural Decision Records

Every major architectural decision is documented here as an ADR.

## Template

Each ADR follows this structure:

1. **Problem** — What needs to be decided
2. **Alternatives** — Options considered
3. **Tradeoffs** — Pros/cons of each alternative
4. **Benchmark Evidence** — Empirical measurements
5. **Decision** — What was chosen and why
6. **Future Revisions** — Conditions that would trigger revisiting
